# 📢 네이버 카페 멀티 알리미 (Naver Cafe Multi Monitor)
* 이 프로그램은 Gemini AI를 이용해 제작된 프로그램입니다. 
  - 해당 대화 링크 :  https://gemini.google.com/share/13eef3feb11a 

### **네이버 카페의 여러 게시판을 동시에 모니터링하여, 새 글이 올라오면 즉시 알람을 울려주는 데스크톱 프로그램입니다.**

## 📝 1. 프로그램 개요
이 프로그램은 사용자가 등록한 다수의 네이버 카페 게시판 URL을 백그라운드(Chrome Headless)에서 독립적으로 감시합니다.  
공지사항을 제외한 **'실제 새 게시글'** 이 감지되면 사용자에게 시각적/청각적 알림을 제공합니다.

### 핵심 기능
* **다중 감시:** 제한 없이 여러 개의 게시판을 목록에 추가하여 동시에 감시할 수 있습니다.
* **설정 자동 저장:** 추가한 항목, 게시판 이름, 볼륨, 감시 주기 등의 설정이 `config.json`에 자동 저장되어 재실행 시 유지됩니다.
* **백그라운드 동작:** 크롬 창을 띄우지 않고 백그라운드에서 조용히 실행됩니다.
* **강력한 커스터마이징:** 항목별로 이름 변경, 알람 볼륨 조절, 감시 주기, 반복 재생 여부를 개별 설정할 수 있습니다.
* **리소스 관리:** 프로그램 종료 시 백그라운드에서 실행 중인 크롬 프로세스를 자동으로 정리하여 메모리 누수를 방지합니다.

## 📖 2. 사용법 (User Guide)

### 준비 사항
* 실행 파일(.exe 또는 .py)과 **같은 폴더**에 알람으로 사용할 **`alarm.mp3`** 파일이 반드시 있어야 합니다.

### 단계별 사용법
1.  **게시판 등록**
  * 네이버 카페에서 감시하고 싶은 **특정 게시판**으로 들어갑니다.
  * 주소창의 URL을 복사하여 프로그램 상단 **'게시판 링크'** 입력란에 붙여넣습니다.
  * **[입력 버튼]**을 누르면 리스트에 항목이 추가됩니다.

2.  **이름 변경**
  * 추가된 항목의 기본 이름(예: 항목 1)을 **클릭**합니다.
  * 입력창이 나타나면 원하는 별명(예: 팬아트 게시판, 거래 게시판)을 입력하고 **엔터**를 누릅니다.

3.  **세부 설정 (우클릭 메뉴)**
  * 리스트의 항목 위에서 **마우스 오른쪽 버튼**을 클릭합니다.
  * **감시 주기 설정:** 10초, 30초, 60초, 5분, 10분 중 선택 (기본 30초).
  * **알람 반복 설정:**
    * `무한 반복`: 알람을 끌 때까지 계속 소리가 납니다.
    * `1회 반복`: 알람이 한 번만 울리고 멈춥니다.
  * **항목 삭제:** 더 이상 감시하지 않는 항목을 리스트에서 제거합니다.

4.  **알람 발생 시 대응**
  * 새 글이 감지되면 해당 항목이 **빨간색**으로 깜빡이며 소리가 납니다.
  * **[알림끄기]** 버튼을 누르면 소리가 멈추고 다시 감시 모드로 돌아갑니다.
  * **볼륨 슬라이더**를 조절하여 해당 항목의 알람 소리 크기를 조절할 수 있습니다.

## ⚠️ 3. 주의 사항 (Precautions)

1.  **메모리 사용량 주의**
  * 이 프로그램은 각 항목마다 **하나의 크롬 브라우저**를 백그라운드에서 실행합니다.
  * 항목을 너무 많이(5개 이상) 추가하면 컴퓨터 메모리(RAM) 점유율이 높아져 PC가 느려질 수 있습니다. **(권장: 3~4개 이내)**
2.  **파일 경로**
  * `alarm.mp3` 파일이 없으면 알람 소리가 나지 않습니다.
  * `config.json` 파일을 삭제하면 저장된 목록과 설정이 모두 초기화됩니다.
3.  **네이버 차단 정책**
  * 감시 주기를 너무 짧게(10초 미만) 설정하거나 동시에 너무 많은 요청을 보내면, 네이버 측에서 일시적으로 접속을 차단할 수 있습니다. (기본 설정인 30초 권장)
4.  **프로그램 종료**
  * 프로그램을 종료할 때는 우측 상단의 **X 버튼**을 눌러 종료해주세요. 그래야 백그라운드에 실행 중인 크롬 프로세스들이 깔끔하게 정리됩니다. (작업 관리자 강제 종료 시 프로세스가 남을 수 있습니다.)

## 📅 4. 개발 연혁 (History)

* **v1.0 ~ v5.0 (초기 개발 단계)**
  * 단일 게시판 감시 기능 구현.
  * Selenium 및 GUI(Tkinter) 기본 구조 설계.
  * 알람 반복/1회 재생 토글 기능 및 오디오 제어 로직 개선.

* **v6.0 (대규모 업데이트)**
  * **다중 감시 시스템 도입:** 여러 게시판을 동시에 리스트로 관리.
  * **데이터 영구 저장:** `config.json`을 통한 설정(항목, 볼륨, 주기 등) 자동 저장/불러오기.
  * **UI 전면 개편:** 스크롤 가능한 리스트 뷰, 항목별 개별 컨트롤, 우클릭 컨텍스트 메뉴 추가.
  * **이름 수정 기능:** 텍스트 클릭 시 수정 가능한 UI 적용.

* **v7.0 (안정성 강화)**
  * **초기화 피드백:** 스레드 접속 성공 시 최신글 번호를 UI에 즉시 반영.
  * **유효성 검사:** 잘못된 URL이나 비정상적인 링크 입력 시 경고창 출력 및 등록 방지.

* **v8.0 (프로세스 최적화)**
  * **좀비 프로세스 방지:** 프로그램 종료 시 실행 중인 모든 ChromeDriver 및 Chrome 프로세스를 명시적으로 찾아 종료하는 안전 장치 추가.
  * 메인 스레드와 감시 스레드 간의 종료 동기화 로직 개선.

* **v9.0 (현재 버전)**
  * **사용자 가이드 탑재:** 프로그램 상단에 **[사용법]** 버튼 추가 및 내장 도움말 팝업 구현.
  * 전체적인 UI/UX 다듬기 및 최종 안정화.
