# 📢 네이버 카페 멀티 알리미 (Naver Cafe Multi Monitor) - API Ver.
* 이 프로그램은 Gemini AI를 이용해 제작된 프로그램입니다.
  - 해당 대화 링크 :  https://gemini.google.com/share/1dc940f1c1c0

**네이버 카페의 여러 게시판을 실시간으로 모니터링하여, 새 글이 올라오면 즉시 알람을 울려주는 고성능 데스크톱 프로그램입니다.**

## 📝 1. 프로그램 개요 (Overview)
이 프로그램은 기존의 무거운 브라우저(Selenium) 방식에서 벗어나, **네이버 공식 API**를 직접 활용하도록 설계되었습니다. 덕분에 매우 적은 메모리 점유율로 다수의 게시판을 동시에 감시할 수 있으며, 사용자 편의를 위한 GUI와 다양한 알람 제어 기능을 제공합니다.

### 🚀 핵심 특징
* **고성능 API 기반:** 크롬 브라우저를 켜지 않아 메모리 사용량이 극도로 적으며 속도가 빠릅니다.
* **다중 감시 지원:** 하나의 프로그램에서 여러 개의 게시판을 목록으로 만들어 동시에 관리할 수 있습니다.
* **설정 자동 저장:** 게시판 목록, 이름, 볼륨, 감시 주기 등의 설정이 `config.json`에 자동 저장됩니다.
* **실시간 제어:** 알람이 울리는 도중에도 볼륨 조절이 즉시 반영되며, 1회 재생/무한 반복 설정이 가능합니다.
* **스마트 URL 파싱:** 네이버 카페의 최신 URL 구조를 자동으로 분석하여 ID를 추출합니다.

## 📖 2. 사용법 (User Guide)

### 🛠️ 준비 사항

**1. Python 라이브러리 설치**
터미널(CMD)에서 아래 명령어를 입력하여 필수 라이브러리를 설치합니다.
```bash
pip install requests pygame
```
**2. 필수 파일 준비**
프로그램 실행 파일(.py 또는 .exe)과 **같은 폴더**에 **`alarm.mp3`** 파일이 반드시 있어야 합니다.

### 📋 단계별 사용 가이드

#### **Step 1. 게시판 등록**
1. 네이버 카페에서 감시하려는 게시판에 접속합니다.
2. 브라우저 주소창의 **URL을 복사**합니다.
  - *지원 형식:* `https://cafe.naver.com/f-e/cafes/{카페ID}/menus/{메뉴ID}`
3. 프로그램 상단 입력창에 붙여넣고 **[입력 버튼]**을 클릭합니다.

#### **Step 2. 항목 관리**
- **이름 변경:** 리스트에 추가된 '항목 1' 등의 텍스트를 클릭하면 이름을 수정할 수 있습니다. (예: 왁물원 팬아트)
- **삭제:** 항목 위에서 **[마우스 우클릭]** → **[항목 삭제]**를 선택합니다.

#### **Step 3. 상세 설정 (우클릭 메뉴)**
항목 위에서 **마우스 오른쪽 버튼**을 클릭하여 개별 설정을 할 수 있습니다.
- **감시 주기 설정:** 10초 ~ 600초 (기본 30초). 너무 빠르면 차단될 수 있으니 주의하세요.
- **알람 반복 설정:**
  - `무한 반복`: 사용자가 꿀 때까지 계속 울립니다.
  - `1회 반복`: 알람이 한 번만 울리고 자동으로 멈추며, 알림 상태도 해제됩니다.

#### **Step 4. 알람 제어**
- **알림 끄기:** 새 글이 감지되어 알람이 울리면 **[알림끄기]** 버튼을 눌러 소리를 멈춥니다.
- **볼륨 조절:** 슬라이더를 통해 각 게시판별 알람 소리 크기를 조절합니다. (재생 중 실시간 반영)

## ⚠️ 3. 주의 사항 (Precautions)

1. **공개 게시판만 가능**
  - 로그인 없이 접근 가능한 '전체 공개' 게시판만 감시할 수 있습니다. '멤버 공개' 게시판은 API 접근 권한 문제로 감시가 불가능합니다.
2. **과도한 요청 주의**
  - 감시 주기를 10초 미만으로 설정하거나, 수십 개의 게시판을 동시에 감시하면 네이버 서버로부터 IP가 일시 차단될 수 있습니다. (권장: 30초 이상)
3. **URL 형식 확인**
  - 구형 URL(`articleList.nhn` 등)은 인식하지 못할 수 있습니다. 최신 네이버 카페 웹 UI의 주소를 사용해주세요.
4. **파일 관리**
  - `config.json` 파일을 삭제하면 저장된 모든 설정과 목록이 초기화됩니다.

## 📅 4. 개발 연혁 (History)

### **[초기 단계] Selenium 기반 (v1 ~ v8)**
- **v1.0:** 웹 브라우저 자동화(Selenium)를 이용한 단일 게시판 감시 구현.
- **v6.0:** 다중 게시판 리스트 관리 및 설정 저장 기능(`config.json`) 도입.
- **v8.0:** 프로그램 종료 시 크롬 좀비 프로세스가 남는 치명적 버그 수정.

### **[성능 혁신] API 전환 (v10 ~ v11)**
- **v10.0:** **Selenium 제거 → Requests API 전환.** 메모리 사용량 90% 절감 및 속도 대폭 향상.
- **v11.0:** API 엔드포인트 변경 (BoardList API v1) 및 정규표현식(Regex)을 이용한 URL 파싱 로직 고도화.

### **[안정화 및 UX 개선] (v14 ~ v15)**
- **v14.0:** 알람 재생 중 볼륨 슬라이더 조작 시 **실시간으로 음량이 변경**되도록 오디오 로직 개선.
- **v15.0 (현재 버전):**
  - **스마트 알람 해제:** '1회 반복' 설정 시, 오디오 재생이 끝나면 자동으로 UI의 알림 상태(빨간색)가 해제되도록 수정.
  - **가이드 탑재:** 프로그램 내 '사용법' 버튼을 통해 상세 가이드 팝업 제공.
